④ Referral system — how attribution works
Referral flow & commission tracking
User A has unique referral code e.g. RAVI2025
→
Shares product link with ref code yoursite.com/product/123?ref=RAVI2025
→
User B clicks link ref code stored in cookie / session
▼
User B registers or logs in Referral code attached to account
→
User B completes purchase
→
System detects ref=RAVI2025 Commission credited to User A's wallet
Technical mechanism: When User B opens the referral link, the ?ref=RAVI2025 parameter is stored in a browser cookie (30-day expiry). If User B registers within 30 days, the referral code is permanently linked to their account. Any purchase by User B — on first purchase or all purchases (configurable) — triggers a commission event for User A. The admin panel controls commission %, validity window, and payout method.
